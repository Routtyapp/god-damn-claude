## UI 디자인 7원칙

사용자가 목표를 달성할 수 있도록 안내하는 핵심 원칙입니다.

### 1. 계층 구조 (Hierarchy)

중요한 정보를 먼저 인식하도록 시각적 단서를 사용합니다.

- **글꼴 크기/두께**: 크고 굵은 텍스트로 핵심 정보 강조
- **대비**: 대조적인 색상으로 시선 유도
- **간격**: 요소 간 관계를 공간으로 표현

> 사용자가 무엇을 먼저 보고, 무엇을 스크롤해야 볼 수 있는지 의도적으로 설계하세요.

### 2. 점진적 공개 (Progressive Disclosure)

한 번에 모든 정보를 보여주지 않습니다.

- 복잡한 프로세스를 단계별로 분리
- 각 단계에서 필요한 정보만 제공
- 현재 진행 상황과 남은 단계를 명확히 표시

### 3. 일관성 (Consistency)

동일한 요소는 동일하게 동작해야 합니다.

- 버튼, 입력, 네비게이션의 패턴 통일
- 불규칙성은 인지 부하를 증가시킴
- 패턴을 벗어날 때는 명확한 이유 필요

### 4. 대비 (Contrast)

중요한 요소로 시선을 끌기 위해 전략적으로 사용합니다.

- **주요 액션**: 강한 대비 (예: 빨간색 삭제 버튼)
- **보조 액션**: 약한 대비 (예: 회색 취소 버튼)

### 5. 접근성 (Accessibility)

모든 사용자가 이용할 수 있어야 합니다.

- 전경/배경 간 충분한 색상 대비
- 이미지에 대체 텍스트 제공
- 키보드 탐색 지원
- 적절한 터치 타겟 크기
- WCAG 2.1 AA 기준 준수

### 6. 근접성 (Proximity)

관련 있는 요소는 가까이 배치합니다.

- 가까운 요소 = 관련 있음
- 먼 요소 = 별개의 기능
- 예: 재생/일시정지/되감기는 함께, 종료는 별도 배치

### 7. 정렬 (Alignment)

일관된 정렬은 질서와 가독성을 만듭니다.

- 그리드 시스템으로 균형 확보
- 예측 가능한 레이아웃으로 탐색 용이
