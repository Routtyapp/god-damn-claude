# Gestalt Principles for Web UI Design
> 웹 UI 설계를 위한 게슈탈트 이론 실전 적용 가이드

---

# 1. 개요

게슈탈트 이론은 인간이 시각 정보를 **개별 요소가 아닌 의미 있는 전체로 인식한다**는 인지 심리학 이론이다.

웹 UI에서는 다음 질문으로 바꿔 이해할 수 있다:

- 사용자는 이 화면을 어떻게 “덩어리”로 인식하는가?
- 무엇이 한 그룹처럼 보이는가?
- 시선은 어디서 시작해 어디로 흐르는가?
- 무엇이 전경이고 무엇이 배경인가?

게슈탈트는 “디자인 감각”이 아니라 **인지 구조 설계 도구**다.

---

# 2. 핵심 원칙과 웹 UI 적용

---

## 2.1 근접성 (Proximity)

### 원칙
가까이 있는 요소는 같은 그룹으로 인식된다.

### UI 적용
- 폼 라벨과 입력 필드는 가까워야 한다.
- 카드 내부 요소는 카드 외부 요소보다 더 촘촘해야 한다.
- 메뉴 항목 간 간격은 그룹 단위로 차이를 둔다.

### 설계 기준
- 그룹 내부 간격 < 그룹 간 간격
- 간격만으로 정보 구조가 드러나야 한다.
- 불필요한 선(border) 사용을 줄이고 여백으로 구조를 만든다.

---

## 2.2 유사성 (Similarity)

### 원칙
색상, 모양, 크기가 비슷하면 같은 그룹으로 인식된다.

### UI 적용
- Primary 버튼은 항상 동일 색상/스타일 유지
- 동일 레벨의 Heading은 같은 타이포 스케일
- 같은 기능의 아이콘은 동일 스타일 사용

### 설계 기준
- 디자인 시스템을 통한 스타일 일관성 유지
- 같은 의미에는 같은 스타일
- 다른 의미에는 명확히 다른 스타일

---

## 2.3 연속성 (Continuity)

### 원칙
사람은 자연스러운 방향과 흐름을 따라 시선을 이동한다.

### UI 적용
- 리스트는 위 → 아래 흐름
- 카드 그리드는 좌 → 우, 위 → 아래
- Z 패턴, F 패턴 레이아웃 활용

### 설계 기준
- 시선 흐름이 끊기지 않도록 정렬 유지
- 불규칙한 배치 금지
- 그리드 시스템을 통한 정렬 강제

---

## 2.4 폐쇄성 (Closure)

### 원칙
불완전한 형태도 인간은 스스로 완성하여 인식한다.

### UI 적용
- 카드에 꼭 모든 면에 border가 필요하지 않다.
- 아이콘은 단순화해도 인식 가능하다.
- Minimal UI 설계 가능.

### 설계 기준
- 과도한 장식 제거
- 핵심 정보에 집중
- 불필요한 시각 노이즈 제거

---

## 2.5 도형-배경 (Figure / Ground)

### 원칙
인간은 전경과 배경을 구분하려는 경향이 있다.

### UI 적용
- 모달은 배경을 어둡게 처리
- 카드에는 배경 대비 혹은 그림자
- CTA 버튼은 배경 대비 강하게

### 설계 기준
- 대비(Contrast) 명확히 확보
- 전경 요소는 시각적으로 분리
- 계층(Hierarchy)을 색/그림자/간격으로 표현

---

## 2.6 공통 운명 (Common Fate)

### 원칙
같이 움직이는 요소는 같은 그룹으로 인식된다.

### UI 적용
- Hover 시 카드 내부 요소가 함께 반응
- 로딩 스켈레톤 애니메이션
- 드롭다운 메뉴 확장 애니메이션

### 설계 기준
- 관련 요소는 동일한 모션 타이밍 사용
- 상호작용 피드백은 일관되게

---

# 3. 게슈탈트와 그리드 시스템

그리드는 근접성과 연속성을 구조적으로 강제한다.

- 12 컬럼 시스템 → 정렬 일관성 확보
- 고정 거터 → 그룹 간 명확한 분리
- 좌우 마진 → 전경/배경 균형 유지
- 4px 스케일 → 리듬감 형성

그리드는 단순한 배치 도구가 아니라
**인지 구조를 통제하는 장치**다.

---

# 4. UI 설계 시 체크리스트

### 구조
- 그룹 내부/외부 간격 차이가 명확한가?
- 시선 흐름이 자연스러운가?

### 스타일
- 같은 의미에 같은 스타일을 사용했는가?
- 불필요한 변형이 존재하지 않는가?

### 대비
- CTA는 충분히 눈에 띄는가?
- 전경/배경이 명확히 구분되는가?

### 복잡도
- 장식 요소가 정보 전달을 방해하지 않는가?
- 간격으로 해결 가능한 문제를 선/박스로 해결하지 않았는가?

---

# 5. 실무에서 자주 발생하는 문제

- 간격이 애매하여 그룹이 모호함
- 버튼 스타일이 화면마다 달라 혼란 발생
- 정렬이 어긋나 시선 흐름 붕괴
- 강조 요소가 너무 많아 우선순위 불명확

이 대부분은 게슈탈트 원칙 위반이다.

---

# 6. 결론

웹 UI는 “요소를 배치하는 작업”이 아니라  
“인지 구조를 설계하는 작업”이다.

게슈탈트 원칙을 이해하면:

- 정보 구조가 명확해지고
- 시선 흐름이 자연스러워지며
- 디자인 일관성이 강화되고
- UX 품질이 안정된다.

디자인 감각은 취향이지만,  
인지 원칙은 과학이다.
