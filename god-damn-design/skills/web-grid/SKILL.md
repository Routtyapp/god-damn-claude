# Web Grid System Specification (4px / 12 Column Standard)

> 목적:  
> 반응형 웹에서 일관된 레이아웃 품질을 유지하기 위한  
> 표준 그리드 설계 규칙 정의 문서

---

# 1. Grid의 4가지 핵심 요소

웹 그리드는 반드시 아래 4가지 요소로 구성된다.

1. **Container (콘텐츠 폭)**
2. **Column Count (단의 수)**
3. **Gutter (단과 단 사이 간격)**
4. **Margin (좌우 마진)**

이 네 가지는 항상 명확하게 정의되어야 하며,
임의로 생략하거나 섞어 쓰지 않는다.

---

# 2. 기본 원칙

## 2.1 4px 스케일 고정

- 모든 수치는 **4px 단위**
- 마진, 거터, 컨테이너, 내부 패딩 모두 4의 배수
- 예외 없음

> 컬럼 너비는 가변이므로 소수점 발생 가능  
> (반응형이기 때문에 허용됨)

---

## 2.2 12 Column 고정

- 모든 반응형 웹은 **12단(12 Columns)** 사용
- 12는 다음과 같이 분할이 유리함:
  - 2
  - 3
  - 4
  - 6

디자인과 개발 양쪽 모두에게 최적화된 구조.

---

# 3. Container (콘텐츠 폭)

## 3.1 선택 가능한 디자인 베이스

디자인은 아래 해상도 중에서 선택해야 한다:

- 1920
- 1600
- 1440
- 1280
- 1200
- 1080
- 1024
- 960

이 범위 내에서 사이트 목적에 맞게 선택한다.

> 임의 수치 사용 금지 (예: 1366, 1180 등 비표준 값 사용 금지)

---

## 3.2 Container 특성

- 고정 값
- 4px 단위
- 브레이크포인트 별로 명확하게 정의

예:

| Breakpoint | Container Width |
|------------|-----------------|
| ≥1440      | 1200            |
| ≥1280      | 1080            |
| ≥1024      | 960             |

---

# 4. Column (단)

## 4.1 Column 수

- 항상 12개
- 반응형에서도 기본 구조 유지

## 4.2 Column Width

- 가변 값
- 소수점 허용
- 자동 계산됨:

Column Width =
(Container Width - (Gutter × (Column Count - 1))) / Column Count


컬럼 값은 고정하지 않는다.

---

# 5. Gutter (단 간격)

## 5.1 허용 값

- 30px
- 24px
- 20px

중 하나 선택.

## 5.2 규칙

- 한 프로젝트 내에서는 기본값 1개 선택
- 특정 영역에서 의도적으로 확장/축소는 허용
- **규칙성 없는 혼합 금지**

예시 (금지):
어떤 섹션은 30
어떤 섹션은 20
어떤 섹션은 10
어떤 섹션은 8


이런 방식은 일관성 파괴.

---

# 6. Margin (좌우 여백)

## 6.1 마진은 필수 요소

좌우 마진을 생략하면:

- 화면이 답답해 보임
- 작은 화면에서 여백이 사라짐
- 가독성 저하

## 6.2 중요한 이유

예:

- Container 1200으로 잘 설계
- 화면이 1200보다 작을 경우
- 좌우 여백이 없다면 콘텐츠가 화면 끝에 붙음

→ UX 품질 급격히 저하

## 6.3 권장

- 브레이크포인트 별 최소 마진 정의
- 모바일에서는 최소 16px 이상 권장 (4px 단위 유지)

---

# 7. 모바일 기준

## 7.1 기본 베이스

### iOS 1x 기준
- 375px

### Android 기준
- 360px

---

## 7.2 표준 선택지

- 414 × 896
- 375 × 812
- 360 × 640

이 세 가지를 기준으로 설계.

---

## 7.3 모바일 그리드 규칙

- 12 컬럼 유지 가능
- 또는 단순화된 4 / 6 컬럼 내부 사용 가능
- 좌우 마진 유지 필수
- 거터는 20px 권장

---

# 8. 반응형 원칙

## 8.1 고정되는 것

- Container Width
- Column Count (12)
- Gutter
- Margin
- 4px 단위 시스템

## 8.2 변하는 것

- Column Width (자동 계산)

---

# 9. 금지 사항

- 4px 단위 위반
- 임의 해상도 베이스 사용
- 컬럼 수 변경 (12 외 사용 금지)
- 거터값 무작위 혼용
- 좌우 마진 생략

---

# 10. 설계 철학

이 그리드는:

- 디자인과 개발의 충돌 최소화
- 계산 가능성 유지
- 반응형 확장성 확보
- 가독성과 안정성 보장

을 목적으로 한다.

---

# 11. 요약

- 4px 기반
- 12 Column
- 고정 Container
- 고정 Gutter (30 / 24 / 20)
- 고정 Margin
- Column Width는 가변
- 모바일 375 / 360 기준

---

이 문서는 프로젝트 초기 단계에서 반드시 합의 후 시작해야 한다.

그리드는 선택이 아니라, 구조다.
